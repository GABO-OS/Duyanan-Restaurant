MODULE 1: USER ACCOUNT MANAGEMENT
================================================================================
Description:
  This module handles all functionalities related to customer accounts on
  the Duyanan Restaurant online ordering system. It covers account creation,
  secure login, and profile maintenance. A valid account is required before
  a customer can place online orders.

--------------------------------------------------------------------------------
FEATURE 1.1 - USER REGISTRATION
--------------------------------------------------------------------------------

USER STORY:
  As a new customer,
  I want to register an account on the Duyanan Restaurant website,
  So that I can log in and place online food orders.

ACCEPTANCE CRITERIA (Given-When-Then):
  SCENARIO 1 - Successful Registration:
    Given I am on the registration page
    When I fill in all required fields (first name, last name, unique email,
         phone, address, and password) and click "Register"
    Then my account should be created successfully
    And I should be redirected to the login page
    And a success message should be displayed

  SCENARIO 2 - Registration with Duplicate Email:
    Given I am on the registration page
    When I enter an email address that is already associated with an
         existing account
    Then the system should reject the registration
    And display an error: "Email is already in use"

  SCENARIO 3 - Registration with Missing Required Fields:
    Given I am on the registration page
    When I submit the form while leaving required fields blank
    Then the system should not proceed with registration
    And validation error messages should appear for each empty required field

--------------------------------------------------------------------------------

USER STORY:
  As a logged-in customer who has placed an online order,
  I want to cancel my order while it is still pending,
  So that I can change my mind before the restaurant starts preparing it.

  SCENARIO 1 - Cancel a Pending Order with a Reason:

    When I click "Cancel Order", provide a reason, and confirm
    Then the order status should change to "CANCELLED"
    And the cancellation reason should be saved

    Given I have a PENDING order
    When I attempt to cancel without entering a cancellation reason
    Then the system should require a reason before allowing cancellation
    And show: "Please provide a reason for cancellation"

FEATURE 4.3 - SUBMIT ORDER FEEDBACK & RATING
--------------------------------------------------------------------------------

USER STORY:
  As a customer who has received a completed online order,
  I want to submit a star rating and a written comment about my experience,
  So that the restaurant can use my feedback to improve their food and service.

ACCEPTANCE CRITERIA (Given-When-Then):
  SCENARIO 1 - Submit Feedback for a Completed Order:
    Given my order status is "COMPLETED"
    When I submit a rating (1 to 5 stars) and a comment
    Then the feedback should be saved and linked to that specific order
    And a success message should be shown

  SCENARIO 2 - Prevent Duplicate Feedback on the Same Order:
    Given I have already submitted feedback for a completed order
    When I attempt to submit feedback again for the same order
    Then the system should reject the second submission
    And display: "Feedback already submitted for this order"

  SCENARIO 3 - Cannot Submit Feedback Without a Comment:
    Given my order is COMPLETED and no feedback has been submitted yet
    When I select a star rating but leave the comment field blank and submit
    Then the system should block the submission
    And show: "Comment is required"
